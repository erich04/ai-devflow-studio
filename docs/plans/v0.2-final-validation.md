# v0.2 Final Validation Plan

## Goal

Make the already implemented v0.2 Final slice release-confirmed by removing the intermittent real
Electron smoke failure and proving the complete verification command is stable.

This validation plan is macOS-local for v0.2. Windows compatibility is a v0.3 acceptance constraint,
but any v0.2 smoke fix must avoid adding macOS-only path, shell, or selector assumptions.

## Code Facts Checked

- `packages/shared/src/domain.ts` already defines `DataOrigin`, `LocalSettings`, and
  `LocalExecutionState` with `settings` and `mcpServers`.
- `apps/desktop/electron/local-store.ts` already uses SQLite schema version `2`, persists local
  settings and MCP servers, and loads a complete local execution snapshot.
- `apps/desktop/electron/main.ts` already exposes IPC handlers for Run creation, Run saving, Agent
  Event saving, settings saving, MCP saving, and local test execution.
- `apps/desktop/src/App.tsx` already separates `seed` and `local` state, persists new Runs, persists
  Gate approval events, persists theme/MCP state, and wires search filtering.
- `scripts/electron-smoke.mjs` already covers real Electron preload security, local project
  selection, theme/MCP persistence, Run creation, Gate approval, local test execution, blocked
  command handling, and restart persistence.

## Current Conflict

The roadmap previously described v0.2 Final as only planned, while code already implements it. The
remaining gap is validation stability, not feature implementation.

The latest full `corepack pnpm verify` run failed once in `scripts/electron-smoke.mjs` when clicking
`通过 Gate`; the button remained disabled after the script tried to select `flow-node-n-design-gate`.
A standalone `corepack pnpm test:electron-smoke` rerun passed. This indicates a flaky smoke
selection/waiting path rather than a missing product capability.
