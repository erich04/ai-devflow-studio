# DevFlow Studio v0.4: Knowledge Governance

## Summary

v0.4 turns the existing fixture-backed Knowledge page into a standards-focused governance slice.
The goal is not real Agent enforcement yet. The goal is to prove that team knowledge can remain in
Git-managed Markdown while DevFlow indexes it, visualizes it, and links workflow evidence back to
standards.

## Scope

- Add a typed Knowledge Source File and Knowledge Document model.
- Index Markdown frontmatter into governance documents, graph entities, and tag relations.
- Add lightweight knowledge references for Runs, Artifacts, Test Evidence, and Gate decisions.
- Add selected-node governance checks in the desktop Inspector.
- Upgrade the Knowledge page to show the Git Markdown index, graph, and current Run references.
- Add representative Markdown files under `docs/knowledge/`.

## Out Of Scope

- No real LLM or Agent enforcement.
- No editable Markdown writer.
- No repository file watcher.
- No vector search or embeddings.
- No remote knowledge API or Postgres persistence.
- No HoneyAI adapter.

## Implementation Notes

- Markdown stays the source of truth.
- The first runtime index is fixture-backed but mirrors real Git paths under `docs/knowledge/`.
- Knowledge logic lives in `packages/shared/src/knowledge.ts` as pure functions.
- Desktop UI consumes shared functions instead of embedding matching rules in React components.
- v0.5 can use the same `KnowledgeGovernanceCheck` shape to block or warn before Gate approval.

## Test Plan

- Unit tests for Markdown indexing, graph generation, reference creation, and governance checks.
- React tests for Inspector governance checks and Knowledge page document/reference display.
- Browser E2E coverage for navigating to Knowledge Governance and searching standards.
- Full verification with:

```bash
corepack pnpm typecheck
corepack pnpm test
corepack pnpm test:e2e
corepack pnpm test:electron-smoke
corepack pnpm build
corepack pnpm verify
```

## Completion Criteria

- A reviewer can see which standards a selected Gate/Node should consider.
- A reviewer can see Markdown source paths and tags in the Knowledge page.
- Current Run references include at least Run, Artifact, Gate decision, and Test Evidence target
  types when evidence exists.
- All existing v0.2/v0.3 local execution and team sync tests remain green.
