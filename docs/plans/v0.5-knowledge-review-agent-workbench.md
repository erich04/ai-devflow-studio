# DevFlow Studio v0.5: Knowledge Review Agent Workbench

## Summary

v0.5 introduces the first real Agent runtime slice in DevFlow Studio: Knowledge Review Agent.
It reads a redacted review context for a selected Run/Node, uses Knowledge Retrieval references,
artifacts, and test evidence, then writes an Agent Review Artifact, Agent Event, Agent Trace,
Gate Advisory, and token/cost usage.

The milestone deliberately implements one Agent well. It does not implement coding Agents,
multi-Agent orchestration, real MCP execution, repository edits, vector RAG, or auto-fix flows.

## Scope

- Shared Knowledge Review Agent Core with deterministic fake provider and OpenAI-compatible
  provider support.
- Electron local runtime using SQLite for review, trace, token usage, artifact, and event
  persistence.
- API backend runtime using the same shared core and Postgres repository boundary.
- Provider credential metadata APIs that never return plaintext secrets.
- Desktop Agent Workbench and Inspector entry point.
- Web manager console display and server-action trigger for backend review.
- Warning-only Gate Advisory by default.
- Redacted Electron summary sync to team API.

## Implementation Progress

- Shared Agent domain types and core are implemented in `packages/shared/src/agent-review.ts`.
- Electron IPC, preload API, local store tables, and main-process runtime are implemented.
- Desktop UI includes an `Agents` navigation view plus Inspector `Agent Review` action.
- API routes include provider list/save, backend Knowledge Review execution, review listing, and
  redacted local review summary sync.
- Seed and Postgres repositories persist provider metadata, Agent Review results, traces, token
  usage, artifacts, and events.
- Web manager console reads Agent review data and can trigger a backend fake-provider review through
  a server action.
- Tests cover shared core, redaction, SQLite persistence, IPC contract, API routes, repository
  boundaries, Web API client/page rendering, browser E2E, and Electron smoke.

## Completion Criteria

- Running Knowledge Review from Desktop Inspector creates review, trace, token usage, artifact, and
  event in local SQLite.
- The Desktop Agent Workbench shows provider status, review history, trace steps, warning-only Gate
  Advisory, and cost source.
- API `/api/agent/knowledge-review` creates team-visible Agent Review data using the same shared
  core.
- Web manager console can view Agent reviews and trigger a backend review.
- Electron summary sync sends only `RemoteAgentReviewSummary`.
- Gate approval remains human-controlled and is not blocked by Agent Advisory in v0.5.

## Future Expansion

- v0.6 can add configurable Gate enforcement policies.
- A future RAG milestone can add embedding/vector provider implementations behind the existing
  Knowledge Retrieval boundary.
- Future MCP work can add real tool-call traces without changing Agent Review persistence.
- Future Agent work can add coding or multi-Agent flows as separate runtimes rather than expanding
  Knowledge Review Agent beyond its review scope.
