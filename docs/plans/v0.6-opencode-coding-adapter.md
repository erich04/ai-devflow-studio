# DevFlow Studio v0.6: opencode Coding Adapter

## Summary

v0.6 turns DevFlow into a DevFlow-native coding workbench without reimplementing opencode. DevFlow
hosts a coding engine, assembles context, creates a managed worktree, relays permissions, archives
diff/bootstrap evidence, and syncs only redacted summaries to the team backend.

## Implementation Progress

- Spike completed: HTTP server selected over direct ACP for the first transport.
- v0.6.0 implemented: fake coding harness, managed git worktree, SQLite schema v4 persistence, UI
  Coding Agent panel, permission approve/reject, fake diff artifact, bootstrap evidence, cancel, and
  worktree open/delete controls.
- v0.6.0 hardening implemented: shared build-task eligibility, DevFlow-native coding brief context,
  runtime publisher/preload subscriptions, permission timeout expiry, and worktree test evidence
  persistence for the fake engine path.
- v0.6.1 foundation implemented: tested opencode HTTP adapter wrapper for session creation,
  permissions, replies, abort, message send, and diff fetch.
- v0.6.1 runtime implemented: env-gated `opencode-http` engine selection, managed `opencode serve`
  process lifecycle, real session/message/permission relay, abort, redacted diff capture, runtime
  dependency bootstrap, and manual `corepack pnpm test:opencode-smoke` entrypoint.
- v0.6.2 implemented: redacted `RemoteCodingAgentSummary` sync endpoint, Electron remote sync
  client support, and Postgres `coding_agent_summaries` read/write path for team overview.

## Remaining Hardening

- Manually sign off `corepack pnpm test:opencode-smoke` on a machine with opencode and provider
  credentials configured.
- Add Windows smoke for managed worktree path handling and dependency bootstrap once v0.6.1 runtime
  is promoted from manual smoke to release validation.

## Guardrails

- No renderer fs/shell access.
- No global opencode auth writes.
- No `--dangerously-skip-permissions`.
- One active Coding Agent Run per Local Project.
- Worktree isolation is not sandboxing.
- Permission timeout rejects by default.
- Remote summary changed paths must be repo-relative and capped at 50 entries.
- Diff artifact patch content is capped at 50,000 characters and added lines are secret-redacted.
