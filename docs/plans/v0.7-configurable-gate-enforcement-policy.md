# DevFlow Studio v0.7: Configurable Gate Enforcement Policy

## Summary

v0.7 upgrades DevFlow from warning-only review to configurable, explainable, auditable Gate
Enforcement. Out-of-box behavior remains warn-only; teams must explicitly apply recommended or
custom policy to block Gate approval.

## Scope

- Shared enforcement policy model, resolver, validator, evaluator, and `canApproveGateNow`.
- Warn-only default policy and Recommended Enforcement Preset.
- Agent Policy Findings in Knowledge Review output.
- API/Postgres policy persistence, evaluation, and Gate override routes.
- Web Team Console policy panel with recommended preset action.
- Electron local policy snapshot storage, override storage, enforcement evaluation IPC, and
  approval write-path enforcement.
- ADR 0010 plus domain-language updates.

## Completion Criteria

- Policy floor clamps weaker project overrides.
- Agent findings can never hard-block.
- Warn-only default preserves human Gate approval.
- Recommended preset blocks deterministic missing-review/testing/API-contract cases.
- Gate approval write paths use `canApproveGateNow`, not renderer-only disabled state.
- Desktop stores policy snapshots and Gate overrides in SQLite schema v5.
- Web can apply the recommended policy through the API boundary.

## Out of Scope

- Automatic remediation or coding retry loops.
- Real MCP policy enforcement.
- Policy signatures/KMS-backed snapshot integrity.
- Full Windows Electron smoke for policy cache.
- In-Desktop team policy editing.
