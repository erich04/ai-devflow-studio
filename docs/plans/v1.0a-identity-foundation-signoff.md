# DevFlow Studio v1.0a: Identity Foundation Signoff

## Summary

v1.0a establishes the identity foundation needed before GitHub OAuth, Desktop pairing, or
self-hosted team deployment. This slice does not add OAuth UI or token exchange. It makes the
existing API/team data model capable of representing authenticated users separately from demo
headers, while preserving the current Team Console projection.

Branch signoff was completed on `devflow-v1.0-team-pilot-foundation` after implementation through
`b92fe01`.

## Scope Completed

- Added shared identity domain types:
  - `User`
  - `AuthAccount`
  - `AuthenticatedIdentity`
  - `DemoSession`
  - `AuthenticatedSession`
- Kept `TeamMember` as a UI/display projection rather than a membership source.
- Kept `project_members` as the project-membership truth.
- Added `auth_accounts` to the team schema and demo seed path.
- Added repository identity lookup:
  - Seed repository resolves demo GitHub auth accounts such as `demo:u-ling`.
  - Postgres repository resolves `auth_accounts -> users -> project_members`.
- Split API session semantics:
  - Missing headers no longer fall back to demo unless `allowDemoFallback: true` is passed.
  - Authenticated sessions require `x-devflow-session-source: authenticated` and
    `x-devflow-auth-account-id`.
- Added authenticated remote-sync header helper for future Desktop pairing.

## Out Of Scope

- GitHub OAuth start/callback/logout.
- First-user organization bootstrap.
- Project creation UI.
- Desktop pairing code and token exchange.
- Invitation or member administration.
- Multi-organization switching.
- Multi-Desktop concurrency guarantees.
