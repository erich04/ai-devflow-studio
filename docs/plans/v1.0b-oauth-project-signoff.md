# DevFlow Studio v1.0b Signoff: GitHub OAuth + Minimal Team Project

## Summary

v1.0b establishes the smallest Web-authenticated team entry point for the Team Pilot Foundation. It
does not add invitations, GitHub App installation, repository permission scanning, Desktop pairing,
or Docker deployment. Those remain in later v1.0 slices.

## Scope Completed

- GitHub identity bootstrap repository flow:
  - Existing GitHub auth accounts resolve to `User`, `AuthAccount`, and `project_members`-backed
    memberships.
  - Empty deployments can bootstrap the first GitHub user as the default organization owner.
  - Existing deployments block unknown GitHub accounts instead of silently promoting them.
- Minimal team project creation:
  - Shared `Project` metadata now includes `slug` and `description`.
  - Team schema v5 persists project `slug` and `description`.
  - `POST /api/team/projects` allows organization owners to create a project and writes
    `project_members` for the creator.
- GitHub OAuth and session boundary:
  - API route resolver supports OAuth start, callback, and logout.
  - Signed `devflow_session` cookies store authenticated session claims.
  - API server resolves session cookies before falling back to explicit demo mode.
  - GitHub OAuth client is env-configured and never stores provider secrets in docs or fixtures.
- Web Team Console entry point:
  - Web API client no longer defaults to `u-erich owner` demo headers.
  - Web API client forwards `devflow_session` cookies when supplied.
  - Team Console links to GitHub OAuth start and exposes a minimal project creation form.
