# DevFlow Studio v1.0d Signoff: Self-Hosted Minimum Deployment

## Summary

v1.0d completes the Team Pilot Foundation deployment slice. DevFlow Studio can now run the minimum
self-hosted stack outside the development shell with API, Web, and Postgres, and the stack has an
explicit Docker smoke that verifies pairing-code exchange, authenticated Desktop sync, Web/API
visibility, and redaction boundaries.

This is still a pilot deployment, not a public SaaS release. It intentionally does not include
automatic HTTPS, Kubernetes, managed cloud hosting, Electron packaging/signing, token rotation UI,
or multi-Desktop concurrency guarantees.

## Delivered

- Added `Dockerfile` and `docker-compose.yml` for the API, Web, and Postgres pilot stack.
- Added `.env.example` with self-hosted configuration placeholders.
- Added API listen-host configuration so containers can bind on `0.0.0.0`.
- Added Web internal/public API URL resolution for container networking.
- Added `scripts/docker-smoke.mjs` and `corepack pnpm test:docker-smoke`.
- Added a GitHub Actions `Docker smoke` job so Docker-dependent validation runs in CI.
- Added the self-hosted pilot guide at `docs/guides/devflow-studio-self-hosted-pilot.md`.
- Updated README, roadmap, and the v1.0 Team Pilot plan with the self-hosted deployment path.

## Notes

- The API container currently runs `tsx src/server.ts` so the workspace TypeScript package boundary
  resolves reliably in the pilot image. A slimmer runtime image with prebuilt shared package
  artifacts remains a future release-engineering task.
- `test:docker-smoke` remains outside default `corepack pnpm verify`; it should be run in CI or on a
  machine with Docker whenever deployment behavior changes.
