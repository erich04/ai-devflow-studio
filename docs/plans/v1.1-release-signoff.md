# DevFlow Studio v1.1.0 Release Signoff

## Summary

v1.1.0 signs off Runtime Cost + Budget Guard on top of the v1.0 team pilot foundation. The release
adds redacted cost summaries for real Coding Agent runtime usage, team-side runtime budget policies,
lead approvals for over-budget runs, and a Desktop pre-provider budget check before paid opencode
work starts.

This is not a public billing release. It does not include SaaS metering, invoices, hosted payment
flows, or automatic live-provider CI.

## Scope Signed Off

- Shared runtime cost estimation and budget decision model.
- Electron Coding Runtime cost estimation before `engine.start(...)`.
- Team API budget policy, budget evaluation, and budget approval endpoints.
- Postgres persistence for runtime budget policies, runtime budget approvals, and redacted coding
  cost summaries.
- Team cost rollups that include redacted coding runtime cost summaries.
- Remote coding summaries with cost and budget-decision metadata.
- Release-only real opencode smoke remains explicit and paid-provider gated.

## Release Procedure

1. Commit the `1.1.0` version and signoff documentation changes.
2. Push `main` and wait for the GitHub Actions `Verify` workflow to pass.
3. Create annotated tag `v1.1.0` on the verified release commit.
4. Push the tag and wait for the GitHub release artifact workflow.

## Known Boundaries

- Budget policy editing and budget approval UI are planned for v1.2.
- If the team budget API is unavailable from Desktop, v1.1 records that no team budget decision was
  applied instead of guessing a policy locally.
- Real opencode provider smoke remains a release-only local signoff path and is intentionally absent
  from default `verify` and GitHub CI.
