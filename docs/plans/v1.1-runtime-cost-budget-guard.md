# DevFlow Studio v1.1: Runtime Cost + Budget Guard

## Summary

v1.1 adds a cost-aware guardrail around real Coding Agent runtime usage. The default verification
path remains deterministic and cost-free through the fake engine, while real provider runs now
produce redacted runtime cost summaries and can be blocked before provider invocation when a team
budget policy requires lead approval.

The core boundary is unchanged: DevFlow stores and syncs summaries, not raw prompts, local paths,
stdout/stderr, patches, or provider secrets.

## Delivered Scope

- Added shared runtime cost estimation for Coding Agent runs.
- Added project-level runtime budget policies and budget approvals.
- Added `BudgetGuardDecision` so Desktop can record whether a run was allowed, warned, blocked, or
  approved over budget.
- Wired Electron Coding Runtime to estimate cost before calling the coding engine.
- Wired Electron to ask the team API budget endpoint before paid provider usage when a non-zero
  estimated cost is present.
- Added API endpoints:
  - `GET /api/runtime/budget-policy`
  - `PUT /api/runtime/budget-policy`
  - `POST /api/runtime/budget/evaluate`
  - `POST /api/runtime/budget-approvals`
- Added Postgres persistence for runtime budget policies, runtime budget approvals, and redacted
  coding cost summaries.
- Included coding runtime cost summaries in team cost rollups.
- Extended remote coding summaries with redacted cost and budget-decision metadata.

## Non-Goals

- No automatic provider shutdown based on live token streaming.
- No hosted billing, invoicing, or SaaS metering.
- No default CI call to a paid model provider.
- No raw prompt, cwd, stdout/stderr, or patch upload.
- No replacement for release-only real opencode smoke; paid live smoke remains explicit.

## Signoff Notes

- Fake/local provider runs are treated as cost-free.
- OpenAI-compatible real provider runs use deterministic token estimation when provider-reported
  usage is unavailable.
- Budget guard blocks before `engine.start(...)`, so an over-budget run does not spend provider
  tokens before lead approval.
- If the team budget API is unavailable from Desktop, the local run is not blocked by a guessed
  policy; the run records that no team budget decision was applied. Team deployments should keep API
  availability monitored before using strict cost controls.
- Real provider release smoke remains explicit:

```bash
DEVFLOW_RUN_OPENCODE_SMOKE=1 corepack pnpm test:opencode-smoke
```
