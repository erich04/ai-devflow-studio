# DevFlow Studio v1.3: Delivery Flow Completion

## Summary

v1.3 moves DevFlow from a strong middle-of-flow workbench to a coherent request-to-delivery loop.
The existing six-stage model already has `clarify`, `design`, `build`, `test`, `pr`, and `accept`
stages, and v0.6-v1.2 already cover Coding Agent, Test Evidence, Gate Enforcement, Agent Review,
budget guard, and team sync. The missing product layer is workflow continuity:

- a real request should create the first workflow state and raw-request artifact;
- Gate approval should advance the current node through `workflow_edges`;
- PR and Acceptance stages should produce delivery artifacts instead of remaining placeholders.

This milestone does not create real GitHub pull requests. v1.3 creates a reviewable PR draft handoff
and an acceptance evidence bundle. v1.4 may turn the draft into GitHub API integration.

## Key Changes

### v1.3.1 Shared Workflow Core

- Add `createWorkflowRunFromRequest(input)` in `packages/shared`.
- Input is title, raw request, project id, creator id, branch name, run id, and timestamp.
- Output includes:
  - a six-stage node template aligned with the existing domain model;
  - workflow edges from clarification through acceptance;
  - a `raw_request` artifact containing the user request;
  - an initial `thinking` event.
- Add `advanceWorkflowAfterGateApproval(input)` in `packages/shared`.
- The helper must:
  - accept only `gate` or `acceptance` nodes;
  - mark the approved node successful;
  - follow the next `workflow_edges` edge;
  - update `currentNodeId`;
  - map run status to the next stage;
  - mark acceptance approval as terminal `completed`.

### v1.3.2 Desktop Request Intake and Write Path

- Change Electron `createRun` IPC from "save a renderer-built run" to "accept request input and let
  main/shared build the run".
- Keep renderer input small: title, request, project id, creator id, branch name.
- Save generated run, artifacts, and event in Electron local state.
- Replace the old hard-coded `approveGate -> building` logic in Electron main with
  `advanceWorkflowAfterGateApproval`.
- Keep browser preview fallback on the same shared helper so UI tests do not carry a parallel
  workflow engine.

### v1.3.3 PR Draft Handoff

- Add `createPrDraftArtifact(input)` in `packages/shared`.
- The draft summarizes:
  - original request;
  - design summary when present;
  - changed paths from Coding Diff Artifacts;
  - latest Test Evidence;
  - Gate Enforcement status;
  - budget decision;
  - Agent Review summary;
  - a safe GitHub compare URL when the project repository mapping is safe.
- Do not include raw patch bodies, raw stdout/stderr, provider secrets, or local absolute paths.
- Desktop Inspector exposes a `生成 PR Draft` action on `kind: 'pr'` nodes.

### v1.3.4 Acceptance Evidence Bundle

- Add `createAcceptanceEvidenceBundleArtifact(input)` in `packages/shared`.
- The bundle references:
  - raw request;
  - PR draft artifact;
  - changed paths;
  - Test Evidence;
  - policy state;
  - budget state;
  - Agent Review summary.
- Desktop Inspector exposes a `生成验收证据包` action on `kind: 'acceptance'` nodes.
- Acceptance still uses the normal Gate Enforcement write path for final approval.

## Non-Goals

- No real GitHub PR creation.
- No automatic push, merge, or branch publication.
- No automatic Gate bypass after generating PR or acceptance artifacts.
- No new database schema.
- No MCP runtime execution.
- No RAG/vector retrieval integration.
- No multi-Desktop concurrency hardening.

## Test Plan

- Unit:
  - request-based workflow creation produces six stages, edges, raw-request artifact, and initial
    event;
  - clarification/design Gate approval advances `currentNodeId` and run status;
  - acceptance approval completes the run;
  - PR draft includes diff/test/policy/budget/review summaries and safe compare URL;
  - unsafe repository mappings omit compare URLs;
  - acceptance bundle references request, PR draft, diff, tests, policy, budget, and review.
- Electron/IPC:
  - `createRun` accepts request input and rejects missing request fields;
  - `saveArtifact` persists generated delivery artifacts;
  - Gate approval write path uses shared advancement semantics.
- React/Desktop:
  - New Run modal sends title/request input;
  - PR node shows `生成 PR Draft`;
  - Acceptance node shows `生成验收证据包`;
  - generated artifacts appear in the Inspector.
- Verification:
  - targeted workflow/App/IPC tests;
  - `corepack pnpm typecheck`;
  - default `corepack pnpm verify` before release signoff.

## Release Criteria

- A user can create a run from a raw request.
- Gate approval advances the workflow instead of forcing every run into `building`.
- PR and Acceptance nodes have concrete artifacts.
- Existing fake-engine verify remains deterministic.
- The walkthrough guide documents the request-to-delivery path and its boundaries.
