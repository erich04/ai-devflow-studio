# Apple-inspired Keynote 视觉风格参考

这份文档记录一组 Apple-inspired 的高级科技视觉原则和提示词，用于 DevFlow Studio 相关的设计探索、PPT、OpenDesign 或图像生成场景。它是视觉参考，不是品牌仿制规范；实际使用时应避免 Apple logo、Apple 官方素材、仿冒官网页面或直接复制 Apple 品牌资产。

## 一句话定义

苹果设计风格不是“白底 + 大标题 + 少文字”这么简单，而是：内容优先、层级清晰、视觉克制、细节精密、软硬件气质统一，并用少量高质量材质、动效和留白制造高级感。

Apple 官方设计入口把 Human Interface Guidelines 定义为“为 Apple 平台创造优秀体验的最新指导与最佳实践”，并提供 Figma/Sketch 设计资源、SF Symbols、Icon Composer 等官方工具，这说明它本质上是一套系统化设计方法，而不只是某种视觉皮肤。

---

## 1. 苹果风格的核心：让内容成为主角

苹果式设计最重要的不是装饰，而是降低界面存在感，让用户先看到内容、产品、结论或行动。在 2025 年 Apple 发布 Liquid Glass 设计时，官方也反复强调 `bring greater focus to content`，也就是让控件、导航和材质服务于内容焦点，而不是抢走注意力。

用于 PPT 时，可以理解为：

每页只讲一个核心观点。
不要把标题、图表、三段文字、五个图标和复杂背景同时塞进一页。苹果式 PPT 的页面通常像一句清晰的发布会金句：先给结论，再给证据。

提示词可以写：

```text
内容优先的 Apple-inspired keynote 风格，每页只有一个核心信息，大面积留白，主标题成为视觉焦点，辅助文字极少，所有图形和装饰都服务于主题，不喧宾夺主。
```

## 2. 清晰层级：大标题、少层级、强对比

苹果风格经常使用非常明确的文字层级：超大标题 + 简短副标题 + 极少正文。Apple 官方字体页面介绍 San Francisco 时强调它提供一致、易读、友好的字形声音，并通过不同字号轮廓和动态字距保证可读性；SF Pro 也是 Apple 平台的系统字体，提供多字重、可变光学尺寸等能力。

PPT 中你不一定要使用 Apple 官方字体，尤其商业或外发材料不要刻意模仿 Apple 字体风格。更实用的做法是用干净、现代、无衬线字体。中文可用苹方、思源黑体、微软雅黑、阿里巴巴普惠体；英文可用 Inter、Helvetica Neue、Arial、Aptos。Apple 系统字体列表中也包含 PingFang SC 等中文字体项，可作为中文无衬线气质的参考。

PPT 实用建议：

| 元素 | 建议 |
|---|---|
| 封面标题 | 48-76 pt，字重 Semibold/Bold |
| 章节标题 | 40-60 pt |
| 正文 | 18-24 pt |
| 注释 | 12-14 pt |
| 层级数量 | 最多 3 层：标题、副标题、说明 |
| 字体风格 | 现代无衬线，少用花体、书法体、装饰字体 |

提示词可以写：

```text
使用现代无衬线字体，标题巨大而克制，字重偏中粗，副标题轻量，正文极少；文字层级清晰，行距舒展，整体具有 Apple keynote 的清晰、安静、专业感。
```

## 3. 留白：不是空，而是制造高级感和注意力

苹果设计的留白通常非常大胆。留白的作用不是“页面没做满”，而是让观众知道哪里最重要。Apple 的 UI Design Dos and Don’ts 也强调清晰、可读、间距、对齐、对比和高分辨率图像等基础原则；比如文本不能重叠，要通过行高或字距提升可读性，并保证文字和背景之间有足够对比。

PPT 中可以这样执行：

| 场景 | 苹果式做法 |
|---|---|
| 封面 | 中间一个大标题，下面一行副标题，背景干净 |
| 数据页 | 一个巨大数字 + 一句解释，不塞满表格 |
| 产品页 | 一张高质量产品图占主视觉，文字退到一侧 |
| 结论页 | 一句话居中，背景极简 |
| 目录页 | 少量条目，间距很大，不用密集编号 |

提示词可以写：

```text
极简构图，大面积负空间，元素之间保持充足呼吸感，页面不拥挤；所有内容严格对齐，版心稳定，视觉焦点明确。
```

## 4. 色彩：中性色为主，少量强调色

苹果风格通常不是靠复杂配色取胜，而是靠黑、白、灰、银、深空黑、浅雾灰这类中性色建立高级感，再用一处强调色引导行动。Apple HIG 的颜色指导强调系统色会适应背景、外观模式和辅助功能设置；这对 PPT 的启发是：色彩要服务可读性和环境，而不是为了炫。

可用于 PPT 的非官方近似色板：

| 用途 | 浅色模式 | 深色模式 |
|---|---|---|
| 背景 | `#F5F5F7` / `#FFFFFF` | `#000000` / `#111111` |
| 主标题 | `#1D1D1F` | `#F5F5F7` |
| 副文本 | `#6E6E73` | `#A1A1A6` |
| 分割线 | `#D2D2D7` | `#2C2C2E` |
| 强调色 | `#0071E3` 蓝 | `#0A84FF` 蓝 |

提示词可以写：

```text
使用高级中性色调，浅灰白背景、深灰文字、少量蓝色作为唯一强调色；整体冷静、克制、科技感强，避免花哨渐变和高饱和撞色。
```

## 5. 材质感：从“纯平面”到 Liquid Glass 的轻盈层次

现在谈苹果风格，不能只停留在 iOS 7 之后的纯扁平感。Apple 2025 年推出的 Liquid Glass 是一种跨平台的新材质，官方描述它会反射、折射周围环境，并根据内容或上下文动态变化；它也被用于按钮、导航、图标、小组件等界面元素。

但 PPT 里不要滥用玻璃效果。Apple 的 Materials 指南也提示，clear Liquid Glass 只适合用于出现在视觉丰富背景上的组件。所以 PPT 里可以用“轻微玻璃卡片”，但不要整页都是磨砂玻璃、反光、透明叠层。

适合使用 Liquid Glass 感的页面：

| 页面类型 | 使用方式 |
|---|---|
| 科技封面 | 半透明信息卡片叠在抽象柔光背景上 |
| 数据仪表盘 | 少量玻璃卡片承载关键数字 |
| 产品路线图 | 半透明胶囊标签和轻微高光 |
| 深色科技页 | 玻璃层 + 柔和光晕 + 精细边框 |

提示词可以写：

```text
加入轻微 Liquid Glass 灵感：半透明磨砂卡片、柔和背景模糊、细腻边缘高光、低强度阴影、层次轻盈；只用于少量重点信息，不覆盖主要文字，不造成可读性下降。
```

## 6. 图像：高质量、真实、干净、产品感

苹果的图像气质通常是：清晰、高分辨率、真实光影、干净背景、主体居中或大面积留白。Apple App Store 营销指南在展示 Apple 产品时也强调真实呈现、避免极端角度、不要改变产品颜色/形状/比例，并建议用清晰、简单的呈现方式。

用于普通 PPT，你可以把这个原则转化为：

| 图像类型 | 苹果式处理 |
|---|---|
| 产品图 | 干净背景，真实阴影，少装饰 |
| 人物图 | 自然姿态，真实场景，不摆拍过度 |
| 抽象图 | 柔和渐变、玻璃质感、低噪点 |
| 图标 | 线性、统一笔画、圆角一致 |
| 截图 | 放进圆角设备框或玻璃卡片中 |

提示词可以写：

```text
使用高分辨率、真实光影的产品级视觉图像，主体清晰，背景干净，构图克制，避免低清图片、过度滤镜、夸张透视和杂乱装饰。
```

## 7. 图标：统一线宽、少而准

Apple 的 SF Symbols 是一个超过 7,000 个符号的系统图标库，设计目标是与 San Francisco 字体无缝结合，并具备多字重、多尺寸、自动与文本对齐等特点。PPT 中不一定使用 SF Symbols，但要学习它的原则：图标要像文字系统的一部分，而不是随机贴纸。

PPT 图标建议：

| 项目 | 建议 |
|---|---|
| 风格 | 线性或轻填充，不混用 3D、手绘、面性图标 |
| 线宽 | 全套统一 |
| 圆角 | 与卡片圆角一致 |
| 数量 | 一页 1-5 个，不要密集铺满 |
| 颜色 | 黑/灰/白为主，少量蓝色强调 |

提示词可以写：

```text
使用统一线宽的极简线性图标，圆角柔和，与文字视觉系统一致；图标数量少，作为辅助解释，不作为装饰堆叠。
```

## 8. 结构：像发布会，而不是报告堆料

苹果式 PPT 更像 keynote 发布会：先情绪，再结论，再证据，再行动。Apple 早期 Human Interface Principles 中强调“审美完整性”不是单纯漂亮，而是外观与功能是否一致；也强调一致性、直接操控、反馈、隐喻和用户控制等原则。转成 PPT，就是每个页面形式要匹配内容目的：数据页就让数据最大，观点页就让一句话最大，流程页就让路径最清楚。

推荐的苹果式 PPT 结构：

| 页码 | 页面类型 | 设计方式 |
|---|---|---|
| 1 | 封面 | 大标题 + 一句副标题 + 极简背景 |
| 2 | 问题 | 一个痛点句子 + 大留白 |
| 3 | 洞察 | 一句话结论 + 抽象视觉 |
| 4 | 方案 | 三个核心模块，卡片式 |
| 5 | 产品/服务 | 大图 + 简短卖点 |
| 6 | 数据证明 | 一个大数字 + 一张简化图表 |
| 7 | 路线图 | 横向时间线，间距大 |
| 8 | 总结 | 金句 + 行动号召 |

## 可直接复制的 PPT 母提示词

下面这段可以直接给 AI PPT 工具、设计师或图像生成工具使用：

```text
请为【主题】制作一套 16:9 高级科技商务 PPT，整体为 Apple-inspired premium keynote style。设计原则是内容优先、极简、清晰、克制、精密、大面积留白。使用浅灰白或深黑背景，现代无衬线字体，超大标题，简短副标题，正文极少。页面元素严格对齐，层级清楚，使用中性色为主，少量蓝色作为强调色。图像采用高分辨率产品级视觉，真实光影，干净背景。可使用轻微 Liquid Glass 灵感：半透明磨砂卡片、柔和背景模糊、细腻边缘高光和低强度阴影，但不要过度。整体感觉应当像苹果发布会 Keynote：安静、自信、高级、未来感、以内容为中心。不要使用 Apple 标志、Apple 官方产品图片、仿冒 Apple 官网页面、花哨渐变、复杂装饰、密集文字、低清素材或杂乱图标。
```

## 页面级提示词模板

### 1. 封面页

```text
设计一页 Apple-inspired keynote 风格 PPT 封面，16:9。背景为浅灰白或深黑，中央放置超大中文标题【标题】，下方一行极简副标题【副标题】。大面积留白，字体为现代无衬线，字重中粗，字距微紧。可加入一个柔和抽象光晕或极简产品级视觉元素，但不能抢标题。整体高级、安静、科技感强、发布会风格。
```

### 2. 章节页

```text
设计一页极简章节过渡页，Apple keynote 风格。页面只有章节编号【01】、章节标题【标题】和一句短说明。使用大面积负空间，标题左下或居中排版，背景为纯净浅灰/黑色，加入极细分割线或微弱玻璃质感。画面克制、清晰、具有高级科技感。
```

### 3. 数据页

```text
设计一页 Apple-inspired 数据展示页。核心是一个巨大数字【数字】，旁边只保留一句解释【解释】。图表极简，线条细，颜色以黑白灰为主，蓝色只用于突出关键数据。不要使用复杂表格，不要堆叠文字。页面应有强烈视觉焦点，像苹果发布会展示性能提升数据。
```

### 4. 产品/方案展示页

```text
设计一页高级产品展示 PPT，Apple-inspired premium product page style。左侧或中央放置高质量产品/方案主视觉，右侧使用大标题【卖点】和 2-3 个简短卖点。背景干净，光影真实，卡片圆角柔和，阴影极轻。整体像高端科技产品发布页，简洁、精准、有呼吸感。
```

### 5. 三点式卖点页

```text
设计一页三大卖点展示页，Apple keynote 风格。标题为【标题】，下方三个圆角卡片，每张卡片包含一个极简线性图标、一句短标题和一行说明。卡片之间间距充足，统一线宽，统一圆角，颜色克制。整体清晰、现代、专业，不使用复杂插画。
```

### 6. 时间线页

```text
设计一页极简路线图 PPT，Apple-inspired。使用横向时间线，分为【阶段1】【阶段2】【阶段3】【阶段4】。每个阶段用圆角胶囊标签和简短说明表示。背景留白充足，线条细，节点轻微发光或使用低透明玻璃质感。整体清晰、轻盈、未来感。
```

### 7. 对比页

```text
设计一页左右对比 PPT，Apple keynote 风格。左侧为“Before/现状”，右侧为“After/目标”。左右两栏极简卡片布局，中间留足间距。使用灰色表达现状，蓝色强调目标。每栏最多 3 条短句。整体清楚、克制、有高级感，不使用红绿强对比和复杂装饰。
```

### 8. 金句总结页

```text
设计一页 Apple-inspired 结尾页。页面中央只有一句大号金句【金句】，下方小字显示行动号召【CTA】。背景为深黑或浅灰白，加入极淡柔光渐变。整体安静、有力量、像发布会最后一页。
```

## 负面提示词：一定要加

为了避免 AI 把“苹果风格”做成廉价仿品，建议在每个提示词后面加：

```text
避免：Apple logo、苹果官网截图、官方产品图片、仿冒 Apple 官网、复杂渐变、赛博朋克霓虹、卡通 3D、廉价玻璃拟态、过强阴影、密集文字、多彩贴纸、随机图标、低分辨率图片、过度装饰、花哨转场、拥挤排版、粗糙对齐、过多颜色、复杂背景纹理。
```

商业或公开场景要尤其注意：Apple 的品牌/渠道身份指南要求相关方不要模仿 Apple 的字体风格，且网站应有自己的 look and feel，不应复制 Apple 网站；所以 PPT 更适合写成 Apple-inspired / 苹果启发式高级科技风，不要写“仿苹果官网一模一样”。
